なぜ関数を学ぶのか

関数は「処理をひとかたまりにする仕組み」

  • 再利用：同じ処理を何度も書かない
  • 見通し：処理の意味が関数名で伝わる
  • 修正容易：変更箇所を1か所に集約できる

利点1：コードの再利用

同じ計算を3回書く → バグが増える

  • 関数なし：同じ式をあちこちに書く
  • 関数あり：1回定義して何度でも呼ぶ

利点2：見通しが良くなる

関数名で「何をしているか」が分かる

例：

  • calculate_bmi(height_cm, weight_kg)
  • celsius_to_fahrenheit(c)
  • calculate_total(price, tax_rate)

利点3：修正が簡単

  • 税率変更／単位変更／仕様変更が入っても → 関数の中だけ直す

保守の基本： 「変更が入る場所を1か所に」

関数の基本構造

関数名(引数1, 引数2, ...) → 戻り値
  • 引数：関数に渡す入力
  • 戻り値：関数が返す出力

関数呼び出しの基本

  • 形：関数名(引数)
  • 返ってきた値を使う：y = 関数名(x)
  • 結果を見る：print(...) / cat(...)

関数の種類

  1. 組み込み関数／ライブラリ関数（すでにある）
  2. ユーザー定義関数（自分で作る）

3.2.1 関数の利用

ねらい：既存の関数を正しく呼び出す

  • 引数（入力）と戻り値（出力）
  • 位置指定／名前指定
  • デフォルト引数
ここからが 3.2.1「関数の利用」です。 既存の関数で、呼び出しの型を身体に入れます。

この後、平方根と対数で練習します。 数学の内容というより、関数の呼び出し方を練習すると思ってください。

サンプル01：平方根（sqrt）

Python

import math
print(math.sqrt(4))  # 2.0

R

print(sqrt(4))       # 2

対数：log₁₀(100) の意味

「10を何乗すると100？」 → 答えは2（10²=100）

  • 常用対数：底10
  • 自然対数：底 e

サンプル02：底を指定して対数

Python

import math
print(math.log(100, 10))  # 2.0

R

print(log(100, base=10))  # 2

位置のみ引数：Pythonの注意点

Rは名前付き指定が柔軟：

R

log(base=10, x=100)  # OK

Pythonの math.log は名前付き指定ができない：

Python

math.log(x=100, base=10)  # エラー

デフォルト引数

底を省略すると？

底を省略すると 自然対数（底 e）

Python

import math
print(math.log(100))  # ln(100)

R

print(log(100))       # ln(100)

デフォルトの検証

$e$を明示する

Python

import math
print(math.log(100))
print(math.log(100, math.e))  # 同じ

R

print(log(100))
print(log(100, base=exp(1)))  # 同じ

専用関数：log10 / log2

読みやすく、誤解が減る

Python

import math
math.log10(100)  # 2
math.log2(1024)  # 10

R

log10(100)
log2(1024)

Rのパイプ演算子

%>% や |>

  • %>%：左の値を右の関数へ（第1引数）
  • |>：Rの標準パイプ（新しめ）

例： 16 %>% sqrt は sqrt(16) と同じ

サンプル：パイプの基本

R

library(tidyverse)

sqrt(16)
16 %>% sqrt

サンプル：パイプの連鎖

R

library(tidyverse)

5 %>% log %>% exp   # 結果は 5 に戻る

関数のドキュメントを読む

  • Python：help(関数) / 関数.__doc__
  • R：?関数 / help(関数)

例：

  • Python：help(math.log)
  • R：?log

よくある落とし穴

  • log(100) を log10 と勘違い
  • 引数の順序を取り違える
  • “動いた”だけで検証しない
  • 表示と計算を混ぜて再利用できない

3.2.2 関数の定義

ねらい：自分で関数を作り、部品化する

  • 入力（引数）を設計する
  • 出力（戻り値）を設計する
  • 小さく作ってテストする

関数定義の基本構文

Python

def f(x, y):
    # 処理
    return result

R

f <- function(x, y) {
  # 処理
  result   # 最後の式が返ることが多い
}

サンプル：引き算関数

Python

def subtract(a, b):
    return a - b
print(subtract(10, 3))  # 7

R

subtract <- function(a, b) {
  a - b
}
print(subtract(10, 3))  # 7

戻り値：計算と表示を分ける

  • 良い：計算は戻り値、表示は呼び出し側
  • 避けたい：関数の中で何でも print する

例： bmi = calculate_bmi(h, w) → print(bmi)

サンプル：温度変換

Python

def celsius_to_fahrenheit(c):
    return c * 9/5 + 32

R

celsius_to_fahrenheit <- function(c) {
  c * 9/5 + 32
}

サンプル：BMI計算

BMI = 体重(kg) / 身長(m)²

Python

def calculate_bmi(height_cm, weight_kg):
    h = height_cm / 100
    return weight_kg / (h ** 2)

R

calculate_bmi <- function(height_cm, weight_kg) {
  h <- height_cm / 100
  weight_kg / (h ^ 2)
}

デフォルト引数（例：消費税）

Python

def calculate_total(price, tax_rate=0.10):
    return price * (1 + tax_rate)

R

calculate_total <- function(price, tax_rate=0.10) {
  price * (1 + tax_rate)
}

無名関数（lambda / function）

短い処理をその場で書く

Python

square = lambda x: x ** 2

R

square <- function(x) x ^ 2

参考：より実践的な書き方

Pythonの発展要素（必要に応じて）

  • 型ヒント：x: float -> float
  • docstring：関数の説明を残す
  • 入力チェック：不正値を早めに弾く
  • 例外処理：try/except で安全に
