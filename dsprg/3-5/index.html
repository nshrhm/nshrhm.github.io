導入クイズ：Aに似ているのは？

次の3つのデータ（3次元ベクトル）を考えます。

記号 ベクトル
A (3, 4, 5)
B (3, 4, 29)
C (9, -18, 8)

質問：Aに似ているのは B と C のどちら？（直感でOK）

類似度と非類似度

非類似度（Dissimilarity）

  • 小さいほど似ている（0が最小：完全一致）
  • 例：ユークリッド距離、マンハッタン距離

類似度（Similarity）

  • 大きいほど似ている（1が最大、-1まで取ることも）
  • 例：コサイン類似度、相関係数

4指標の全体像

指標 種類 直感 よく合う場面
ユークリッド距離 非類似度 直線距離 位置・量そのもの
マンハッタン距離 非類似度 縦横の移動 スパース／外れ値に比較的強い
コサイン類似度 類似度 向き（角度） テキスト・埋め込み
相関係数 類似度 増減パターン 時系列・波形

キーワード：目的に合わせて“似ている”を定義する

ユークリッド距離（Euclidean）

2点間の直線距離

$$ d(x,y)=\sqrt{\sum_{i=1}^{n} (x_i-y_i)^2} $$

ユークリッド距離の注意点

  • 外れ値に敏感（差を二乗するため）
  • スケール（単位）に依存
    • 例：cmとkgを混ぜると距離が歪む

実務では標準化など前処理を検討

例（ユークリッド）

A=(3,4,5), B=(3,4,29), C=(9,-18,8)

  • $d(A,B)=24$
  • $d(A,C)=23$

結論：ユークリッド距離では C が近い

実装（ユークリッド：Python / R）

from scipy.spatial import distance
A = [3,4,5]; B = [3,4,29]; C = [9,-18,8]
print(distance.euclidean(A, B))
print(distance.euclidean(A, C))

A <- c(3,4,5); B <- c(3,4,29); C <- c(9,-18,8)
sqrt(sum((B-A)^2))
sqrt(sum((C-A)^2))

マンハッタン距離（Manhattan）

碁盤目状に移動する距離（タクシー距離）

$$ d(x,y)=\sum_{i=1}^{n}|x_i-y_i| $$

例（マンハッタン）— 結論が逆転

  • $d(A,B)=24$
  • $d(A,C)=31$

結論：マンハッタン距離では B が近い

同じデータでも指標で最近傍が変わる

実装

マンハッタン：Python / R

from scipy.spatial import distance
print(distance.cityblock(A, B))
print(distance.cityblock(A, C))

sum(abs(B-A))
sum(abs(C-A))
  • 外れ値の影響を少し抑えたい／スパースな特徴量で有利なことがある

コサイン類似

Cosine similarity

ベクトルの 向き（角度） の近さ

$$ \cos(\theta)=\frac{x\cdot y}{|x|,|y|} $$

角度で読む（直感）

  • 同じ方向（$\theta\approx0^\circ$）→ 1
  • 直交（$\theta=90^\circ$）→ 0
  • 逆方向（$\theta=180^\circ$）→ -1

「大きさ」より「パターン（方向）」を重視

例（コサイン）

  • cos(A,B)=0.8170
  • cos(A,C)=-0.0327

結論：コサイン類似度では B が似ている

実装（コサイン）— 距離に注意

from scipy.spatial import distance
# distance.cosine は「距離」= 1 - 類似度
sim_AB = 1 - distance.cosine(A, B)

library(proxy)
sim_AB <- simil(A, B, method = "cosine")
  • Pythonでは「距離」を返す関数があるので注意

相関係数（Pearson correlation）

2つのデータの線形な関係の強さ（ここでは類似度として使う）

$$ r=\frac{\sum_{i}(x_i-\bar x)(y_i-\bar y)}{\sqrt{\sum_i(x_i-\bar x)^2}\sqrt{\sum_i(y_i-\bar y)^2}} $$

相関とコサインの関係

  • 相関係数は、
    1. ベクトルを平均との差に変換し
    2. その上でのコサイン類似度に相当

原点基準（コサイン）と平均基準（相関）の違い

例（相関）

  • $r(A,B)$=0.8825（強い相関）
  • $r(A,C)$=-0.0327（弱い相関）

結論：相関係数では $B$ が似ている

実装（相関：Python / R）

from scipy.spatial import distance
from scipy.stats import pearsonr

r_AB = 1 - distance.correlation(A, B)  # 1-相関距離
r2, p = pearsonr(A, B)                 # 相関係数とp値

cor(A, B)
  • distance.correlation は「距離」なので 1 から引く

距離行列・類似度行列

  • データが増えるとペア数は急増
    • 10個 → 45ペア
    • 100個 → 4950ペア

行列にする

  • 対称行列
  • 対角：距離なら0、類似度なら1

データフレームの置き方

原則：1つの対象 = 1行、特徴量 = 列

x y z
A 3 4 5
B 3 4 29
C 9 -18 8

一括計算（Python：cdist）

import pandas as pd
from scipy.spatial import distance

df = pd.DataFrame({
  "x":[3,3,9], "y":[4,4,-18], "z":[5,29,8]
}, index=["A","B","C"])

D_eu  = distance.cdist(df, df, metric="euclidean")
D_man = distance.cdist(df, df, metric="cityblock")
S_cos = 1 - distance.cdist(df, df, metric="cosine")
S_cor = 1 - distance.cdist(df, df, metric="correlation")

一括計算（R：proxy）

library(proxy)

df <- data.frame(
  x=c(3,3,9), y=c(4,4,-18), z=c(5,29,8),
  row.names=c("A","B","C")
)

D_eu  <- proxy::dist(df, method="euclidean")
D_man <- proxy::dist(df, method="manhattan")
S_cos <- proxy::simil(df, method="cosine")
S_cor <- cor(t(df))  # 行同士の相関

統合演習：都市の気候データ

  • 都市ごとの月平均気温（12ヶ月）をベクトルとして比較

対象

  • 東京、札幌、那覇、ロンドン、シドニー

タスク

  1. 4指標の行列を計算
  2. 東京の最近傍を指標ごとに求める
  3. 理由を言語化する

演習の実装イメージ（Python）

# climate_df: 行=都市、列=1〜12月（12次元）
D_eu  = distance.cdist(climate_df, climate_df, "euclidean")
D_man = distance.cdist(climate_df, climate_df, "cityblock")
S_cos = 1 - distance.cdist(climate_df, climate_df, "cosine")
S_cor = 1 - distance.cdist(climate_df, climate_df, "correlation")

# 東京の最近傍を探す（距離は最小、類似度は最大）

結果：東京の最近傍（要点）

  • ユークリッド距離：ロンドン
  • マンハッタン距離：ロンドン
  • コサイン類似度：ロンドン
  • 相関係数：札幌

指標によって結論が変わる

なぜ相関は札幌？

  • 相関は「平均との差」で**形（波形）**を比較
  • 東京と札幌は
    • 夏に高く、冬に低い
    • 季節変動のパターンが似ている
  • ロンドンは絶対値として近い月が多く、距離で有利になりやすい
