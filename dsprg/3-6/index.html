3.6.1 Rのパッケージ管理

R：パッケージとライブラリ

  • パッケージ：関数やデータのまとまり（例：ggplot2, dplyr）
  • ライブラリ：パッケージが置かれている場所
  • library(ggplot2)＝“ggplot2を使える状態にする”

R：インストールと呼び出し

  1. インストール：install.packages("tidyverse")
  2. 呼び出し：library(tidyverse)
  3. 使用：dplyr::filter(...) など

library()の基本

library(tidyverse)
  • tidyverseは複数パッケージの集合
  • 読み込むと、使える関数が増える

tidyverseの“conflict”（衝突）

例：filter()が複数のパッケージにある

  • dplyr::filter()（データ操作）
  • stats::filter()（時系列フィルタ）

対処：どちらの関数か明示する

典型エラー：パッケージが見つからない

library(tidyverse)
# Error: there is no package called 'tidyverse'

意味：まだインストールされていない

install.packages()の基本

install.packages("tidyverse")
library(tidyverse)
  • インストールは 文字列
  • 呼び出しは 名前（通常は引用符なし）

まとめて入れる（実務でよくやる）

install.packages(c("dplyr", "ggplot2", "readr"))
  • c()でベクトルにして渡す
  • 環境構築では“まとめて”が基本

::（名前空間）を使う

high_scorers <- dplyr::filter(my_scores, math >= 90)
  • library()なしでも関数を呼べる
  • 衝突回避にも強い

環境を確認する（トラブル時の必須）

  • packageVersion("dplyr")：バージョン確認
  • sessionInfo()：環境情報（授業で重要）
  • .libPaths()：ライブラリの場所

更新・削除（必要なときだけ）

  • 更新：update.packages(ask = FALSE)
  • 削除：remove.packages("パッケージ名")

GitHubから入れる（CRANに無いとき）

# 例：GitHub上のパッケージ
# remotes::install_github("tidyverse/ggplot2")
  • 研究開発版・未公開版を使う場面
  • 依存関係やバージョン差に注意

よくあるRトラブル（対処の型）

  • 00LOCKが残る → ロックディレクトリ削除
  • 権限が無い → ライブラリ場所を確認（.libPaths()）
  • ミラーが遅い/落ちる → repos設定を見直す

  • 「there is no package called ...の原因と対処を説明して」
  • 「dplyr::filterとstats::filterの違いを教えて」

3.6.2 Pythonの
モジュール管理

Python：モジュール/パッケージ/ライブラリ

  • モジュール：.pyファイル1つ（例：math）
  • パッケージ：複数モジュールのまとまり（フォルダ構成）
  • ライブラリ：機能の大きな集合（NumPy, pandasなど）

標準ライブラリ vs 外部ライブラリ

  • 標準ライブラリ：最初から入っている（例：math, random）
  • 外部ライブラリ：自分で入れる（例：numpy, pandas）

ポイント：外部は pipでインストール が必要

importの4パターン（まずは覚える）

  1. import numpy
  2. import numpy as np
  3. from numpy import mean
  4. from numpy import *（非推奨）

importの使い分け（おすすめ）

目的 推奨 理由
一般的 import numpy as np 慣習・短い
特定関数だけ from math import sqrt 明確
学習用（慎重に） import numpy 明示的

pipで入れる（外部ライブラリ）

pip install numpy pandas matplotlib
pip list
pip show numpy
  • pip = Pythonのパッケージ管理ツール
  • インストール済み確認もできる

どこでpipを打つ？（WSL2 + venv）

$ cd /home/datasci/work
$ venvc          # 仮想環境を起動
(class) $ pip install numpy
(class) $ python my_script.py

典型エラー

  • ModuleNotFoundError
import numpy
# ModuleNotFoundError: No module named 'numpy'

対処

  1. 仮想環境を確認
  2. pip install numpy

3.6.3 主要
ライブラリの実践

NumPy：数値計算の基本

  • 配列（ndarray）で高速計算
  • 代表関数：np.mean, np.sum, np.max, np.min
  • 統計量をまず出してみる

NumPy 実装イメージ

import numpy as np

x = np.array([10, 20, 30, 40, 50])
print(np.mean(x), np.max(x), np.min(x))

pandas：表データ（DataFrame）

  • Excelの表に近い構造
  • 代表操作：作成 / フィルタ / 集計（groupby）
  • shape, columns, dtypes をまず見る

pandas 実装イメージ

import pandas as pd

df = pd.DataFrame({"name": ["A","B"], "score": [80, 95]})
print(df)
print(df[df["score"] >= 90])

R tidyverse：パイプでデータ処理

  • %>%（または|>）で処理をつなぐ
  • filter, select, mutate, summarise, group_by
  • 「読みやすいデータ処理」を目指す

tidyverse 実装イメージ

library(tidyverse)

df <- tibble(name = c("A","B"), score = c(80, 95))

df %>%
  filter(score >= 90)

同じ処理をRとPythonで（比較）

  • 目的：「やりたいこと」→「言語ごとの書き方」
  • 例：グループ別合計、並べ替え、上位表示

  • 「Rのtidyverseで同等処理に書き換えて」
