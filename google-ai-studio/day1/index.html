第1回: 起動できたら勝ち

AI Studioで動く画面を作る

講師：白濵 成希

下関市立大学 データサイエンス学部

今日から、「AIと対話してWebアプリを作ろう」という講座を始めます。第1回のタイトルは「起動できたら勝ち」です。今日は、完成度の高いアプリを作ることが目標ではありません。Google Chromeを開き、Google AI Studioにログインし、Buildで日本語のプロンプトを入力して、簡単な画面とボタンが動くところまで確認します。ここまでできれば、第1回としては十分成功です。

今日のゴール

Google AI Studioで、画面とボタンを作る。

できれば成功

  • Google AI Studioにログインできる
  • Buildを開ける
  • 日本語でプロンプトを入力できる
  • ボタンを押すと画面が変わる

今日のゴールはとてもシンプルです。Google AI Studioに入り、Buildを開き、日本語で「こんな画面を作ってください」と頼みます。そして、画面にタイトルやボタンが出て、ボタンを押したときに表示が変われば成功です。今日は、保存機能やログイン機能、公開機能までは進みません。まずは、AIと対話して画面が作れるという感覚をつかみます。

今日の大事なルール

  • 講座用のプライベートGmailで作業する
  • 画面にメールアドレスが出たら注意する
  • 課金・公開・APIキーには進まない
  • 分からない画面が出たら止まる

今日は「AI Studio上で動く」ところまで。

最初に大事なルールを確認します。この講座では、原則として講座用のプライベートGmailを使います。勤務先や学校のGoogleアカウントでは、管理者設定によりAI Studioが使えない場合があります。また、画面にはメールアドレスが表示されることがあります。画面共有やスクリーンショットでは、個人情報が映らないように注意します。今日は、課金設定、アプリ公開、APIキー作成には進みません。分からない画面が出たら、そのまま進まず講師に確認してください。

今日の流れ

  1. Chromeを起動する
  2. Google AI Studioにログインする
  3. Buildを開く
  4. 最初のプロンプトを入力する
  5. 生成された画面を確認する
  6. 小さく修正する
  7. 今日できたことを確認する

今日の流れです。まずChromeを起動します。次にGoogle AI Studioにログインします。ログインできたら、左側のメニューからBuildを開きます。Buildでは、作りたい画面を日本語で説明します。そのあと、生成された画面を確認し、ボタンを押して動作を見ます。最後に、追加の指示で少しだけ見た目を直します。今日は、ここまでできれば十分です。

1. Chromeを起動する

今日使う入口は Google Chrome です。

確認すること

  • Chromeが開いている
  • 画面上部のアドレスバーが見える
  • URLを直接入力できる

では、最初にGoogle Chromeを起動します。画面下にChromeのアイコンがある人は、それをクリックしてください。見つからない人は、スタートメニューから「Chrome」と検索してください。Chromeが開いたら、画面上部の細長い入力欄を見ます。ここがアドレスバーです。次の手順では、このアドレスバーにGoogle AI StudioのURLを直接入力します。検索して探すこともできますが、今日は迷子を減らすため、URLを直接入力する方法で進めます。

2. Google AI Studioにアクセスする

アドレスバーに入力します。

https://aistudio.google.com/

入力したら Enter を押します。

Chromeのアドレスバーをクリックしてください。そこに、https://aistudio.google.com/ と入力します。入力できたらEnterキーを押してください。Google検索欄に「AI Studio」と入力して探してもたどり着く場合はありますが、今日は全員が同じ場所に行けるように、URLを直接入力します。URLを入力すると、Google AI Studioのページが開きます。

3. 講座用Gmailでログインする

確認すること

  • 講座用のプライベートGmailを使う
  • 勤務先・学校・家族のアカウントでは進めない

Googleアカウントのログイン画面が出る場合があります。その場合は、講座用のプライベートGmailでログインしてください。すでにログイン済みの人は、そのままAI Studioが開くこともあります。複数のアカウントが表示される人は、どのアカウントを使うか確認します。勤務先、学校、家族のアカウントで進まないようにしてください。また、画面の右上や左下にメールアドレスが表示されることがあります。

4. AI Studioに入れたか確認する

入れていればOK

  • 左上に Google AI Studio が見える
  • 左側に Playground が見える
  • 左側に Build が見える

ログインできると、Google AI Studioの画面が表示されます。画面左上にGoogle AI Studioと出ていれば、まずは大丈夫です。左側のメニューにPlaygroundやBuildが見えると思います。最初はPlaygroundが開いている場合がありますが、それは間違いではありません。AI Studioに入れています。今日は、このあとBuildを使います。

5. Buildを開く

左側メニューから Build を選びます。

Buildは、AIと会話しながらWebアプリを作る場所です。

次に、左側メニューのBuildをクリックします。Buildは、AIと会話しながらアプリを作るための場所です。ここでは、最初からプログラムを書く必要はありません。「こういう画面を作ってください」と日本語で説明すると、AIが画面やコードを作ってくれます。今日は、コードを直接編集することは扱いません。日本語で頼んで、プレビューで確認するところに集中します。

6. Build画面の確認

見つけるもの

  • アプリの説明を入れる入力欄
  • 送信ボタン
  • 周辺に出る提案ボタン

注意: Add database and auth は今日は押しません。

Build画面が開いたら、中央付近にアプリの説明を入れる入力欄があります。ここに、作りたいアプリの内容を書きます。周辺には、提案ボタンや連携ボタンが表示されることがあります。その中にAdd database and authのようなボタンが見える場合がありますが、今日は押しません。これはデータベースやログイン機能に進む導線です。第1回では使いません。

7. 最初のプロンプト

初心者向けの簡単なWebアプリを作ってください。
画面には、次の要素を入れてください。

- タイトル：「はじめてのAIアプリ」
- 説明文：「ボタンを押すとメッセージが変わります」
- ボタン：「押してみる」
- ボタンを押すと、「できました！」と表示される

デザインはシンプルで、文字を大きめにしてください。
まだログイン機能、データ保存、Firebase、APIキー、外部サービス連携は使わないでください。

ここで、最初のプロンプトを入力します。この文章をそのままコピーして、Buildの入力欄に貼り付けてください。内容は、タイトル、説明文、ボタン、ボタンを押した後のメッセージだけです。今日は、まだログイン機能、データ保存、Firebase、APIキー、外部サービス連携は使わない、と明記してあります。AIに頼むときは、作りたいものだけでなく、今日は使わないものも書いておくと、余計な方向に進みにくくなります。

8. プロンプトの読み方

このプロンプトで伝えているこ

  • 何を作るか
  • 画面に何を置くか
  • ボタンを押すと何が起きるか
  • 今日は何を使わないか

プロンプトは、AIへの指示文です。ただし、難しく考える必要はありません。作りたい画面を、できるだけ具体的に説明すれば大丈夫です。今回のプロンプトでは、タイトル、説明文、ボタン、押した後のメッセージを指定しています。さらに、ログイン、データ保存、Firebase、APIキー、外部サービス連携は使わない、と指定しています。これは、第1回の範囲を守るために大事です。

9. 送信して待つ

プロンプトを貼り付けたら送信します。

生成中は少し待ちます。

  • 画面が変わるまで待つ
  • 途中で別のボタンを押さない
  • エラーが出たら講師に確認する

プロンプトを貼り付けたら、送信ボタンを押します。AIがアプリを作り始めるので、少し待ちます。画面が変わるまで、別のボタンをいろいろ押さないようにしてください。生成には時間がかかることがあります。もしエラーが出たり、支払い方法やAPIキーのような画面が出たりした場合は、先に進まず講師に確認してください。

10. 生成結果を確認する

まず見るところ

  • タイトルがあるか
  • 説明文があるか
  • ボタンがあるか
  • ボタンを押した後の表示があるか

見た目が違っても、まずはOKです。

生成が終わったら、作られた画面を確認します。まず、細かい見た目ではなく、必要な部品があるかを見ます。タイトルがあるか、説明文があるか、ボタンがあるかを確認してください。文字の大きさや色、配置が隣の人と違っていても大丈夫です。AIが作るものは、毎回少し変わります。今日は、画面が出て、ボタンがあることが大事です。

11. ボタンを押してみる

「押してみる」 をクリックします。

成功の目安

  • 「できました！」が出る
  • または、何かメッセージが変わる

では、ボタンを押してみましょう。「押してみる」というボタンをクリックします。ボタンを押して、「できました！」と表示されれば、とても分かりやすい成功です。少し違う言葉でも、ボタンを押したあとに何か表示が変われば、まずは成功と考えてください。今日は、プログラムの中身を読むことより、画面が反応することを確認します。

12. うまくいかないとき

失敗ではなく、AIに相談する材料です。

よくある状態

  • ボタンがない
  • 押しても変わらない
  • 画面が複雑すぎる
  • エラーが出た

うまくいかない場合もあります。ボタンが出ていない、ボタンを押しても何も変わらない、画面が複雑になりすぎた、エラーが出た、ということがあります。しかし、これは失敗ではありません。AIに相談する材料です。この講座では、エラーや想定と違う結果を、AIにもう一度説明して直していく練習をします。

13. 困ったときのプロンプト

ボタンを押しても表示が変わりません。
「押してみる」ボタンをクリックしたら、
画面に「できました！」と表示されるように修正してください。
ログイン機能、データ保存、Firebase、APIキーは使わないでください。

ボタンを押しても表示が変わらない場合は、このようにAIに伝えます。大事なのは、今起きていることと、直してほしいことをはっきり書くことです。「ボタンを押しても表示が変わりません」と状況を説明し、「クリックしたら、できました！と表示されるように修正してください」と依頼します。そして、今日使わないもの、つまりログイン機能、データ保存、Firebase、APIキーは使わない、ともう一度書きます。

14. 小さく修正する

AIにもう一度お願いして、画面を整えます。

修正の例

  • タイトルを大きくする
  • 文字を中央に置く
  • ボタンを押しやすくする
  • メッセージを見やすくする

次に、できた画面を少し修正します。AIでアプリを作るときは、一度で完成させるよりも、作って、見て、直す流れが大事です。今日は、見た目を少し整えるだけにします。タイトルを大きくする、中央に配置する、ボタンを押しやすくする、メッセージを見やすくする、といった小さな修正をAIに依頼します。

15. 修正プロンプト

画面をもう少し見やすくしてください。
次のように修正してください。

- タイトルを大きくする
- タイトルと説明文を中央ぞろえにする
- ボタンを少し大きくする
- ボタンの下に、結果メッセージが分かりやすく表示されるようにする

ログイン機能、データ保存、Firebase、APIキー、外部サービス連携は使わないでください。

この修正プロンプトを入力します。ここでは、タイトルを大きくする、中央ぞろえにする、ボタンを大きくする、結果メッセージを見やすくする、という修正だけを頼んでいます。機能は増やしません。ログイン機能、データ保存、Firebase、APIキー、外部サービス連携は使わない、と書いています。第1回では、見た目とボタンの反応だけに集中します。

16. 直ったか確認する

確認すること

  • 画面が少し見やすくなったか
  • ボタンが押しやすくなったか
  • ボタンを押すと反応するか
  • 機能が増えすぎていないか

修正が終わったら、もう一度プレビューを確認します。タイトルが大きくなったか、中央に配置されたか、ボタンが押しやすくなったかを見ます。そして、ボタンをもう一度押して、反応するか確認します。もしAIが勝手にログイン機能やデータ保存機能を追加してしまった場合は、今日はそこには進みません。機能が増えすぎていないかも確認します。

17. 今日は押さないもの

見えても押しません。

  • Deploy / Publish
  • Get API key
  • Add database and auth
  • Upgrade
  • Set up billing / Link billing account
  • Upgrade to Blaze

ここで、今日は押さないものを確認します。DeployやPublishは、公開につながる可能性があります。Get API keyは、APIキー作成につながります。Add database and authは、データベースやログイン機能の追加につながります。Upgrade、Set up billing、Link billing account、Upgrade to Blazeは、課金やプラン変更につながる可能性があります。これらの表示が見えても、今日は押しません。出てきた場合は、講師に確認してください。

今日やらないことをもう一度確認します。第1回では、アプリ公開、APIキー作成、Firebase、Firestore、Googleログイン、Cloud Billing、クレジットカード登録、外部サービス連携は扱いません。今日は、Google AI Studioの中で、簡単な画面とボタンが動くところまでです。ここをしっかり区切ることで、安全に講座を進められます。

21. 第1回 終了チェック

以下を押していない

Deploy / Get API key / Add database and auth / Upgrade

最後に、第1回の終了チェックです。Google Chromeを起動できた、Google AI Studioにログインできた、Buildを開けた、プロンプトを入力できた、画面にタイトル、説明文、ボタンが出た、ボタンを押して反応を確認できた。ここまでできていれば、第1回は成功です。追加の修正指示を1回試せた人は、さらに良いです。Deploy、Get API key、Add database and auth、Upgradeは押していないことも確認してください。個人情報がスクリーンショットに映っていないことも確認します。
