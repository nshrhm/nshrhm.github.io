Google Cloud Starter Tierについて説明します。

Starter Tierを利用できる場合は、自分でGoogle CloudプロジェクトやCloud Billingアカウントを設定せず、Google AI StudioからWebアプリを公開できます。公開できるサービスは最大2個です。

ここで、FirebaseのSparkプランと混同しないようにします。Google Cloud Starter Tierは、AI StudioからWebアプリを公開するための経路です。Firebase Sparkプランは、AuthenticationやFirestoreをFirebase側の無償枠で使うための料金プランです。役割が違います。

Starter Tierが表示されたからといって、FirebaseをBlazeプランへ変更する必要はありません。この講座では、Firebase側は引き続きSparkプランの範囲を維持します。

Starter Tierにも上限があり、すべてのアカウントが利用できるわけではありません。そのため、今日は受講生に一斉公開をさせません。

Gemini APIの費用上限を設定する画面

  • アプリ内Geminiを使う場合の費用管理
  • 1か月に利用できる金額の上限を設定する
  • 上限に達するとGemini APIの利用が一時停止
  • この講座では講師が管理し、受講生は変更しない

みなさん、この画面は、アプリ内でGeminiを使う場合に、1か月の費用上限を設定する画面です。

前の画面で Set spend cap を選ぶと、この画面が表示されます。 入力欄には、1か月に利用できる費用の上限額を日本円で入力します。 金額を入力して Save を押すと、その上限が設定されます。
