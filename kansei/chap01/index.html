なぜ今「感性データ処理」なのか

機能価値が中心の見方

  • 使える
  • 速い
  • 壊れにくい

今、差が出る体験価値

  • 心地よい
  • 安心できる
  • 高級感がある / かわいい
  • 没入できる / 疲れない
  • 自分に合っていると感じる

この体験としての価値を扱うのが「感性」です。

デジタル化で「感性」は扱いやすくなった

  • 行動ログ: クリック、滞在、離脱、リピート
  • 主観評価: アンケート、感性ワード、自由記述
  • 生理・身体指標: 心拍、視線、皮膚電気反応 など

重要なのは、単に数字を集めることではなく、

  • 何を感じるものとして定義するか
  • どのデータで表すか
  • 分析結果をどう意思決定につなげるか

感性とは何か

感性：モノやコトに対する刺激を感覚器官で受けた時に、 印象や感情を抱く心の働き

  • 主観的である
  • 曖昧である
  • ただし、工学的アプローチで定量化できる

要点：感性は「気分一般」ではなく、対象への反応として現れる

感性は「対象 × 人 × 文脈」で変わる

対象 人 文脈
同じ BGM 集中したい人 / リラックスしたい人 図書館 / カフェ / 自宅
同じ UI 初心者 / 熟練者 急いでいるとき / 余裕があるとき

したがって、感性を扱うときは平均値だけで終わらず、

「誰が」「どんな状況で」感じたのかを意識する必要があります。

感性データとは何か

種類 例 強み 注意点
主観評価 5段階評価、SD法、VAS 感じ方そのものに近い 質問の作り方で変わる
言語データ 感性ワード、自由記述、レビュー 体験のニュアンスが出る ばらつき・解釈が難しい
行動ログ クリック、滞在、購買、離脱 大量・客観的に取れる なぜそう感じたかは直接わからない
生理・身体指標 心拍、視線、皮膚電気反応 無意識の反応に近い 計測条件・倫理・解釈が難しい

この授業では、まず 主観評価（アンケート） を軸に考えます。

感性データ処理とは何か

感性データ処理：感性データの分析・可視化・解釈を行い、 製品設計やサービス改善に活用するプロセス

基本の流れ

  1. 対象（製品・UI・体験）
  2. → 刺激提示（使う / 見る / 触れる）
  3. → 言語化・数値化（アンケート、SD法、VAS など）
  4. → データ分析（平均・分散・相関・多変量解析）
  5. → 解釈（印象の軸、群差、特徴）
  6. → 活用（設計変更、改善、検証）

この授業で後に扱う代表的な評価法

方法 何を測るか 例 扱う回
SD法 言葉による印象の軸 明るい–暗い、洗練–野暮ったい 第8回
VAS 感じ方の強さ 快適でない–非常に快適 第11-13回

今日の段階での理解

  • SD法：言葉の軸で印象を整理する
  • VAS：強さの軸で微妙な差を表す

初回では、「感性を測る方法には種類がある」 と分かれば十分です。

感性工学とは何か

感性工学：生活者が製品やサービスに対して抱く「感性」を把握し、 それを設計・改善に活かす学問・技術の体系

  • 心理・デザイン・工学・情報が交わる
  • 「感じ方」を設計へ返すための枠組み
  • 本講義の中心である感性データ処理は、 測る・整理する・解釈する部分を担う

次回の主題：感性工学の成り立ち、意義、実務での使われ方

産業応用の見取り図

分野 対象 感性・主観体験 データと活用
自動車・交通 EV の警告音 危険度、気づきやすさ、不快感 主観評価から音設計を検討
家電 掃除機の動作音 不快さ、静かさ、力強さ 心理音響指標 + 主観評価で比較
UI / UX Web デザイン案 美しい、明るい、信頼できる、分かりやすい 感性ワード + SD法 → 改善仕様
ゲーム バイオフィードバック 緊張、ストレス、没入 生理反応で難易度を動的調整
医療・ヘルスケア PARO との触れ合い 痛み、落ち着き、幸福感 VAS + 生理指標で介入効果を見る
