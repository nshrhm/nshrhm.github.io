なぜ今「感性工学」なのか

基本性能が満たされやすい市場 差が出る体験価値
使える 触って 気持ちいい
速い 見て 高級感がある
壊れにくい 使って 迷いにくい
機能差だけでは選ばれにくい 聴いて 不快ではない / 自分に合っている

これらは数値化しにくく個人差も大きい一方で、選ばれる理由 に強く関わります。

「感性」を設計の言葉に翻訳する

感性工学で見る観点

  • どの対象が
  • どんな人に
  • どんな状況で
  • どんな印象や感情を生むのか

例： 「安心できる」は何から生まれるか

  • 色や形の落ち着き
  • 表示の分かりやすさ
  • 音の大きさや質感
  • 触ったときの手触り
  • ブランドへの既有知識

要点：感性工学は、関係を「なんとなく」で終わらせず、設計の言葉に翻訳する

ミニ活動1：身近な製品は、どんな「感じ方」を狙っていそうか

  1. Think（2分）

    • 最近使った製品・サービスを1つ選ぶ
    • ユーザーに感じてほしそうなことを 2つ 書く
    • 例：安心、軽快、かわいい、落ち着く、信頼できる

  2. Pair（2〜3分）

    • 2人で共有する
    • その感じ方を生みそうな設計要素を 1つ 挙げる

  3. Share（2分）

    • 全体共有し、「感じ方」と「設計要素」が結びつきうることを確認する

感性工学とは何か

feeling や image を設計要素へ翻訳する技術

本講義での捉え方

  • 人が対象に対して抱く印象・感情・体験価値を捉える
  • それを 設計・改善 に結びつける
  • 分析のための分析ではなく、設計のための方法論 として考える

感性工学が扱う問い

  • どんな人に、どんな感じ方をしてほしいのか
  • その感じ方は、どの言葉で表せるのか
  • その感じ方は、どの設計要素に関係していそうか
  • 実際に評価すると、どの要素が効いているのか
  • その知見を、どう設計変更に返すのか

ポイント：「感じ方を測る」こと と 「設計へ返す」こと を切り離さない

感性工学の基本プロセス

  1. 対象を決める 何を改善・設計したいのか
  2. 狙う感じ方を定める 安心感、高級感、楽しさ など
  3. 言葉や評価方法を決める 感性ワード、質問紙、実験条件 など
  4. 評価してデータを集める 主観評価、行動ログ、生理指標 など
  5. 設計要素との関係を読む どの要素がどの感じ方に結びつくかを整理する
  6. 設計へ反映する 色、形、素材、音、レイアウト、手順を見直す

要約：狙う感じ方 → 評価・データ化 → 設計要素との対応づけ → 改善案の作成

成り立ちと広がり

節目 意味
1970年代以降 日本で感性工学の考え方が体系化される
1996年 古典的整理が広く参照される
1998年 日本感性工学会（JSKE）設立
2007年 KEER 会議シリーズ開始
2020年代 デジタル技術、オンライン調査、AI、レビュー分析との接続が進展

現在は、製品だけでなく サービス、UX、VR、医療、教育 などへ対象が広がっています。

事例を読むときの見方

  1. 対象 何を評価・設計したのか
  2. 感性・主観体験 何を「感じるもの」として扱ったのか
  3. データと活用 どう測り、どう設計へ返したのか

まずはこの3点で読む と、分野が違っても事例の骨格が見えやすい

産業応用の概観

分野 対象 感性・主観体験 データと活用
パッケージ サフラン包装 情緒的印象、好ましさ 要素整理と評価から設計提案へつなげる
照明・空間 壁付け照明 夜の雰囲気、昼間の見え方の魅力 要素-印象関係を整理し、照明設計に使う
製品音 掃除機の動作音 不快さ、音の印象 心理音響指標 + 聴取実験で音設計に活かす
外観 × レビュー分析 家庭用ポータブルエアコン 外観に対する感性語 レビューから語を抽出し、評価モデルに活用する
HCI / UX コンピュータマウス emotional user response、使い心地 リモートUX調査で主観データを収集する

事例から見える共通パターン

  • まず 狙う感じ方 を言葉にする
  • 次に 設計要素 を候補として整理する
  • 評価実験やレビュー、主観評定で データを集める
  • その関係を根拠に 設計変更や改善提案 を行う

核心：「感じ方」と「設計要素」の関係を、議論できる形にすること

ミニ活動2：事例を「設計の言葉」に言い換える

  1. Think（3分）

    • 表の事例から1つ選ぶ

    • 次の3点を一文ずつで書く

      1. 狙っている感じ方
      2. それに関係しそうな設計要素
      3. 次に確かめるべきこと

  2. Pair（3〜4分）

    • 互いのメモを見せ合い、つながりが自然かを確認する

  3. Share（2〜3分）

    • 同じ対象でも、狙う感じ方によって見るべき要素が変わることを確認する

分析の前に、何をどう測るかを設計する

データサイエンスの見え方

  • 説明変数（X）：色、形、素材、音、配置などの設計要素
  • 目的変数（y）：安心感、高級感、楽しさなどの感性評価

ただし難しさがある

  • そもそも 何を感じ方として測るのか を決める必要がある
  • 同じ言葉でも、人によって意味がずれる
  • 文脈や経験によって評価が変わる
  • 質問の仕方や提示順序でも結果が変わる

だから：分析の前に 「何を、どう測るか」 を設計する必要がある
