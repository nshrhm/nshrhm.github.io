なぜ今「評価」を学ぶのか

体験価値を議論するには、
「感じ方」を言葉とデータにする必要 がある。

  • 基本機能だけでは差がつきにくい製品・サービス、体験の質が重要

  • (例)以下のような印象はそのままでは比較が困難、設計にも返しにくい

    • 「心地よい」「高級感がある」「安心できる」「不快ではない」「自分に合っている」

  • ひとつの体験は、しばしば 複数の感覚 の組み合わせで成立する

  • したがって、後の分析より前に 評価設計 が重要になる

評価設計で決めること

評価設計は、後の分析で扱うデータの定義になる。

  • 何を「感じるもの」として扱うか
  • どの感覚に注目するか
  • どんな言葉や項目で尋ねるか
  • 主観だけで取るか、行動や生理も補助的に使うか

ミニ活動1

最近の体験を「どの感覚が効いていたか」で見る

  1. Think（2分） 最近「使いやすい」「心地よい」と感じた製品・サービスを1つ選ぶ
  2. Think（続き） その体験に関わっていそうな感覚を挙げる
  3. Pair（2〜3分） 2人で共有し、特に効いていそうな感覚を1つ選んで理由を話す
  4. Share（1〜2分） 全体で数例共有し、同じ対象でも注目する感覚が割れることを確認する

正解探しではない
感じ方を感覚に分解して考えること が目的

五感を「刺激の入口」として整理する

感覚 着目する特徴 例
視覚 色、形、明るさ、動き、配置 UI、パッケージ、ポスター、空間
聴覚 音量、音高、音色、リズム 通知音、機械音、BGM、操作音
触覚 硬さ、粗さ、温度、重さ、振動 スマホ背面、ボタン、素材感
嗅覚 香り、におい 店舗空間、化粧品、食品
味覚 甘味、酸味、塩味、苦味、うま味 飲料、食品、薬

  • 第3回では、五感を 「どの感覚から刺激が入るか」を考える入口 として用いる
  • 厳密には知覚はもっと多様だが、導入としては五感で整理すると見通しがよい
  • まずは 対象のどの感覚的特徴が体験に効いていそうか を押さえる

多感覚統合：体験は組み合わせで変わる

  • 多感覚統合 複数の感覚情報が組み合わさり、ひとつの体験として知覚されること
  • クロスモーダル ある感覚が、別の感覚の感じ方を変えること

身近な例

  • 同じ飲み物でも、BGMで味の印象が変わる
  • 同じ画面でも、操作音や振動で「反応が良い／悪い」の印象が変わる
  • 同じ空間でも、照明や香りで「落ち着く／落ち着かない」が変わる

評価実験では、単独で切り出すか／利用状況に近い組み合わせで提示するか で、結果の意味が変わる。

感性ワードとは何か

曖昧な印象を測れる形に
近づけるための言葉

  • 「かわいい」「落ち着く」「軽快だ」
  • 「高級感がある」
  • 「信頼できそうだ」
  • 「爽快だ」

感性ワードの役割

感性ワードは、印象を評価項目へ
変換するための足場になる。

  • 後の 評価項目 の材料になる
  • 後の 質問文 の材料になる
  • 「何を感じたか」を受講者・被験者・設計者の間で共有しやすくする

感性ワードの基本手順

1. 集める

  • インタビュー、自由記述、レビュー、広告文、既存研究

2. 整理する

  • 類似語をまとめる
  • 意味が近すぎる語を整理する

まずは候補を広く出し、
似た意味の語をまとめる。

感性ワードの基本手順

3. 絞る

  • 対象に合っているか
  • 回答者が理解しやすいか
  • 後で評価項目にしやすいか
  • 似た役割の語が重なりすぎていないか

「思いついた語を並べる」だけでなく、
目的に合う語へ絞り込む。

ミニ活動2

感性ワードを「集める → 整理する」

  1. Think（3分） 「スマホのホーム画面」または「学食の雰囲気」を対象に、印象語を8個程度書く
  2. Pair（3〜4分） 互いの語を見比べ、意味が近い語をまとめる
  3. Share（2〜3分） 全体で「残したい語」と「まとめてよさそうな語」を共有する

見るポイント

  • 対象に合っている語か
  • 似た語が重なりすぎていないか
  • 後で質問項目にしやすい語か

主観評価と客観評価

系統 例 分かること 強み 注意点
主観評価 評定、順位づけ、自由記述 本人がどう感じたか 感じ方を直接たずねられる 質問文や文脈の影響を受ける
行動データ クリック、操作時間、視線遷移、エラー数 迷い、探索、行動のしやすさ 実際の行動として残る 感情や印象そのものは直接は分からない
生理データ 心拍、皮膚電気反応、脳波など 緊張、覚醒、負荷の変化の手がかり 言語化しにくい反応を補える 機材・倫理・解釈が難しい

  • 客観データの方が常に正しいわけではない
  • 基本は 主観評価を中心 に、必要に応じて行動・生理を補助的に組み合わせる
  • 重要なのは「どれが上か」ではなく、それぞれが何を示し、何を直接は示さないか を見極めること

今回の射程

  • 五感は評価設計の入口になる
  • 体験は多感覚で成立する
  • 感性ワードには整理の手順が必要である
  • 主観評価と客観評価は役割が違う

事例からみる「評価」の使われ方

事例 対象 主に関わる感覚 主なデータ 今日の観点
食体験デザイン 音楽・環境音と飲食体験 聴覚 × 味覚 官能評価、主観評定 多感覚統合の例
パッケージ評価 サフラン包装や製品外観 視覚 感性ワード、印象評価、設計要素との対応づけ 感性ワードを設計へ返す例
製品音評価 掃除機の音 聴覚 主観評価 + サイコアコースティクス指標 主観と物理指標の併用例
Web/UI評価 WebインターフェイスのA/B比較 視覚 主観評定 + 視線計測 + 専門家評価 主観／客観の組み合わせ例

分野が違っても、どの感覚に注目し、どんな言葉やデータで取るか という設計の考え方は共通している。

事例から見える共通パターン

  1. 対象を決める 何を評価するかを明確にする
  2. 感じ方を定義する どんな印象や体験を扱うか決める
  3. データの取り方を選ぶ 主観、行動、生理のどれを使うか考える
  4. 設計へ返す どの要素を変えると感じ方が変わりそうかを読む

ミニ活動3（8〜10分）

次のうち1つを選び、主観評価で1つ／客観データで1つ 候補を挙げる。

  • このアプリは信頼できそうか
  • この掃除機の音は不快ではないか
  • このパッケージは高級感があるか

次回への橋渡し

同じ感じ方でも、取り方でデータの型が変わる

取り方 データの型
自由記述で書いてもらう 言語データ
5段階で答えてもらう 順序的な評定データ
0〜100の線上で答えてもらう 連続量に近いデータ
クリック数や操作時間を測る 行動ログ

  • 何を感じるか だけでなく、どう取るか によって使える分析方法が変わる
  • 後半で扱う平均、相関、多次元尺度法、主成分分析、因子分析、検定は、データの型と無関係ではない

次回のキーワード

  • 質的変数／量的変数
  • 名義尺度・順序尺度・間隔尺度・比率尺度
  • 「平均してよいか」をどう考えるか

どの分析が使えるかは、
どう測ったか に依存する。

まとめ

  • 感性評価の入口は、まず どの感覚が効いているか を整理すること
  • 実際の体験は、しばしば 多感覚統合 として成立すること
  • 感性ワードは、感じ方を測れる形に近づけるための重要な材料であること
  • 主観評価と客観評価は対立ではなく、役割が違う こと
  • 次回は、ここで得たデータを 変数の種類と尺度水準 の観点から扱うこと

今回の要点は、評価法の詳細を覚えることではなく、評価を設計する視点をつくること。

確認問題

  1. 五感をすべて挙げ、それぞれが関わりそうな評価対象を1つずつ書きなさい。
  2. 多感覚統合とは何か。身近な例を1つ挙げて説明しなさい。
  3. 感性ワードを絞るときの基準を3つ書きなさい。
  4. 主観評価、行動データ、生理データの違いを簡潔に説明しなさい。
  5. 「このWebサイトは安心できそうか」を評価したいとき、主観評価と客観データの候補を1つずつ挙げなさい。

授業後課題

身近な対象を「感性評価の入口」から整理する

  • 分量：A4 1枚程度（目安 600〜800字）
  • 対象：普段使う製品・サービスを1つ選ぶ

必須で書くこと

  1. その対象で効いていそうな感覚を 2〜3個
  2. その対象を表す感性ワード候補を 6〜8個
  3. そのうち特に重要そうな感性を 2つ
  4. それを確かめるためのデータ候補を、主観評価で1つ、客観データで1つ
