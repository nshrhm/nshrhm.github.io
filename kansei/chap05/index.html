「似ている」は何で決まるか

観点 例
見た目 このアプリは、あのアプリに似ている
目的 このサービスは、別のサービスと使う場面が近い
操作感 この UI は、他の UI と流れが似ている
雰囲気 この店や車は、別の対象に近い印象がある

同じ「似ている」でも、基準は1つではない。

MDS とは何か

多次元尺度法（MDS）

  • 対象の近さ／遠さをもとに、2次元や3次元の空間に配置して構造を可視化する手法

基本の対応

  • 入力：対象間の類似度・非類似度・距離
  • 出力：対象を配置したマップ（2D / 3D）

地図の読み方の出発点

  • 近い点どうしは「似ている」
  • 遠い点どうしは「似ていない」

距離行列の考え方

類似度と非類似度

  • 類似度：値が大きいほど似ている
  • 非類似度（距離）：値が大きいほど似ていない

距離行列（非類似度行列）の基本

  • 正方行列である
  • 対角成分は 0（自分と自分の距離）
  • 左右対称である

距離行列の例

4つの音刺激 A〜D の非類似度

A B C D
A 0 2 7 6
B 2 0 6 5
C 7 6 0 3
D 6 5 3 0

読み取り

  • A と B は近い
  • C と D も近い
  • A と C は遠い

距離の集め方：直接法と間接法

方法 何をするか 例 強み 注意点
直接法 人に「似ている／違う」を直接たずねる 一対比較、ソーティング、順位づけ 感じた近さをそのまま入れやすい 対象数が増えると負担が大きい
間接法 属性データから距離を計算する SD法平均、ログ、物理量から距離計算 対象数が多くても扱いやすい どの属性を使うかで結果が変わる

直接法の負担

一対比較の数

k(k−1)/2

  • 6 個 → 15 通り
  • 10 個 → 45 通り
  • 20 個 → 190 通り

ミニ確認

  • 対象が 6 個、10 個、15 個のときの比較数を計算する
  • 例題を、直接法か間接法かに分ける

計量MDSと非計量MDS

種類 何をできるだけ保とうとするか 向きやすいデータ
計量MDS 距離の量そのもの 間隔尺度・比率尺度に近い距離データ
非計量MDS 距離の順序関係 「A は B より C に近い」などの順序的データ

  • 第4回で学んだ尺度水準が選択に関わる
  • 感性評価の「似ている／似ていない」は、順序的に扱う方が自然な場面が多い
  • そのため、非計量MDSは心理的距離データと相性がよい

ストレスと次元数

  • ストレス（stress） 元の距離関係と、地図上の距離関係のずれを表す指標
  • 小さいほど、元の距離関係をよく再現している
  • 次元数を増やすと、一般にはストレスは下がる
  • ただし、次元数が増えるほど地図は読みにくい

大事なのは、ストレス最小だけではない。 再現性と読みやすさのバランスをとることが重要である。

MDS の地図をどう読むか

まず見ること

  1. 近い点どうしを見る
  2. 離れた点や端の点を見る
  3. かたまりがありそうかを見る
  4. 軸の意味を後から言語化する

MDS の地図を読むときの注意

解釈の注意

  • 軸は最初から「高級感」「使いやすさ」と決まっているわけではない
  • 回転や反転に本質的な意味はない
  • 原点や軸の単位に過度な意味を置かない

  • A と B が近い

小さな例で読む MDS：解釈

読めること 例
近い点 A と B は近い印象のグループ
別のかたまり C と D も別の近い印象グループ
軸らしさ 例：落ち着いた ↔ 派手

  • MDS は軸を自動で命名しない。
  • 解釈は、対象に関する知識と文脈をもって行う。

MDS が効く事例①：デザイン・空間

対象 感性・主観体験 データと活用
自動車外観デザイン スポーティー、上質、親しみやすいなどの印象 印象評価項目の類似関係を整理し、評価項目の選定やセグメント理解に活かす
触覚インタフェースの形状 押す、選ぶ、増減するなどの機能的印象 形状どうしの関連を可視化し、直感的な haptic UI 設計に活かす
オフィス内装の写真 温かい、落ち着く、明るい、親しみやすい印象 写真の印象類似を地図化し、木質内装と非木質内装の印象差を整理する

MDS が効く事例②：音・味覚

対象 感性・主観体験 データと活用
音刺激（音色・音質など） この音は、あの音に似ている／違うという知覚的類似 心理実験の非類似度データから、音知覚の主要な次元を探索する
コーヒーの苦味 「苦い」の中の下位品質の違い グループ化と配置の結果を地図化し、代表刺激の選定や追加実験設計に活かす

共通点

主観的な近さを地図にし、設計・分類・評価の議論に使う

MDS と他手法の違い

手法 入力 何を保とうとするか / 何を見るか 典型的な出力
MDS 対象 × 対象の距離 対象間の近さ・遠さ ポジショニングマップ、知覚マップ
PCA 対象 × 変数のデータ表 分散の大きい方向 主成分軸
数量化Ⅲ類 / 対応分析 対象 × カテゴリーのクロス表 行と列の関係 同時布置図

  • MDS は、対象どうしの関係が最初から与えられているときに強い
  • 予測よりも、構造理解・探索的分析に向く

MDS を使うときの注意点

  • 結果は、距離行列の質に強く依存する
  • 直接法では、参加者が何を基準に「似ている」と答えたかが重要
  • 間接法では、どの属性で距離を作ったかが重要
  • 複数人データでは、平均するか、個人差を見るかを考える必要がある
  • 「ボタンを押せば客観的な地図が出る」と考えない

距離の設計そのものが、分析設計である。

まとめ

  • MDS は、対象どうしの距離から地図を作る手法である
  • 距離の集め方には、直接法と間接法がある
  • 感性評価では、非計量MDSが自然な場面が多い
  • ストレスは再現性の指標だが、読みやすさとのバランスが重要である
  • 地図は、近い点・かたまり・端の点・軸らしさを読みながら解釈する
  • 軸は自動で意味づけされない
