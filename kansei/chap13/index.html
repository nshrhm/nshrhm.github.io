回答者として守ること

  • 他の人と相談せず、個人の感じ方として回答する

回答内容は成績評価の対象ではない。

また、数字表示を原則オフにしているため、何点を狙うのではなく、直感的な位置で答えます。刺激が再生できない、表示されないなどの問題があれば、その情報はデータ品質の確認に必要です。

横長表と縦長表

CSVの形は、分析しやすい形に整える。

可視化や集計では、縦長表が扱いやすい。 差分計算では、横長表が扱いやすい。

データには、横長表と縦長表があります。横長表は1人1行で、A_Q1、B_Q1のように列が分かれています。差分を計算しやすい形です。

縦長表は、1行が1つの評定値になります。グラフを作るときや、刺激ごと、項目ごとに集計するときに便利です。Pythonでは、必要に応じてこの2つの形を行き来します。

今回の主分析：刺激Aと刺激Bを比べる

各VAS項目について、次の問いを立てる。

  • Q1：刺激Aと刺激Bで、評定値に違いがあるか
  • Q2：刺激Aと刺激Bで、評定値に違いがあるか
  • Q3：刺激Aと刺激Bで、評定値に違いがあるか

重要

同じ人がAとBの両方を評価しているので、

B − A の差分を回答者ごとに作る。

今回の中心は、刺激Aと刺激Bの比較です。3つのVAS項目があるので、項目ごとにAとBを比べます。

同じ人がAとBの両方を評価しているため、回答者ごとにBからAを引いた差分を作ります。差分が正ならBの方が高い、負ならAの方が高い、というように方向を決めて読みます。

今回は、分析を広げすぎない

今回の授業内プロジェクト
標準仕様 2刺激 × 3 VAS項目 に固定

したがって

  • 属性別比較は原則行わない
  • 複雑な多変量解析は行わない
  • まず、各VAS項目で刺激A・Bの差を見る

一般的なVAS分析では、3条件以上の比較や相関分析を行うこともあります。しかし、今回の授業内プロジェクトでは、みなさんの混乱を避けるために2刺激、3項目に固定しています。

したがって、主分析は刺激Aと刺激Bの対応あり比較です。相関や属性別分析は、必要がありデータが十分な場合の補助的分析に留めます。

データを見る前の仮説を確認する

第12回で書いた仮説に戻る。

例

  • 刺激Aは刺激Bより、落ち着きが高いと予想した
  • 刺激Bは刺激Aより、活発さが高いと予想した
  • 好ましさは方向を決めず、探索的に比較するとした

今日やること

  • 仮説と同じ方向で結果を読む
  • 予想外の結果も記録する
  • データを見た後で、主分析項目を変更しない

分析の前に、第12回で立てた仮説を確認します。仮説は正解を当てるためのものではありませんが、データを見る前の考えを固定する役割があります。

データを見てから、都合のよい項目だけを主分析にするのは避けます。予想と違う結果が出た場合は、それ自体を考察の材料にします。

3つのVAS項目をどう扱うか

3項目は、同じ重みで何でも平均しない。

まず行うこと

  1. 各項目を別々に集計する
  2. 各項目で刺激A・Bを比較する
  3. 第12回の主分析項目を最初に読む
  4. 残り2項目は補助的に読む

注意

  • 3項目を勝手に合計点にしない
  • 似た項目でも、意味が違えば別々に読む
  • どの項目が主分析かを明記する

今回の3項目は、必ずしも同じ尺度の下位項目ではありません。たとえば、好ましさ、落ち着き、活発さは、それぞれ異なる構成概念です。

したがって、勝手に3項目の平均点を作って総合スコアにするのは避けます。まずは項目ごとに読みます。第12回で主分析項目を決めていた場合は、それを最初に解釈します。

各アンケートについて、次の表を作る。

  • ばらつきは大きいか
  • 中央値と平均が大きくずれていないか

最初に作るのは記述統計の表です。刺激Aと刺激B、項目Q1からQ3の組合せで、n、平均、標準偏差、中央値、四分位範囲を出します。

この段階では、p値を急いで出す必要はありません。まず、どちらが高いか、ばらつきは大きいか、平均と中央値が大きくずれていないかを確認します。

各VAS項目について、刺激A・Bの分布を比べる。

向いている図

  • バイオリンプロット
  • ヒストグラム

見ること

  • 0や100に集まっていないか
  • 中央付近に集中していないか

VASデータでは、平均だけを見ずに分布を確認します。0や100に多く集まる場合、床効果や天井効果が考えられます。中央に集中している場合、刺激差が弱い、質問が曖昧、回答者が迷ったなどの可能性があります。

外れ値らしい値があっても、すぐに削除してはいけません。まずは刺激や回答状況を確認し、レポートでは外れ値をどう扱ったかを明記します。

今回の標準設計では、同じ人がAとBを評価する。

  • AとBを結ぶスロープグラフ

  • 一部の人だけが大きく変化していないか

対応ありのデータでは、AとBの分布を別々に見るだけでは不十分です。同じ人の中でどれくらい変わったかを見る必要があります。

AとBを線で結ぶ図や、BマイナスAの差分分布を見ると、多くの回答者が同じ方向に動いているのか、一部の回答者だけが強く反応しているのかが分かります。

縦長表があると、刺激と項目でグループ化して平均や標準偏差を出しやすくなります。配布Notebookでは、実際のCSVからこの縦長表を作る処理を使います。

この段階で、各刺激と各項目の回答数がそろっているかを確認してください。ある項目だけ回答数が少ない場合は、欠損や整形ミスがないか見直します。

Python：差分を作る

同じ回答者のAとBを対応づける。

# 例：1つのVAS項目だけを取り出す
item_id = "Q1"

sub = long_df[long_df["item"] == item_id]

wide = sub.pivot(
    index="respondent",
    columns="stimulus",
    values="value"
)

wide["diff_B_minus_A"] = wide["B"] - wide["A"]

display(wide.head())

差分の意味

  • diff > 0：Bの方が高い
  • diff < 0：Aの方が高い

差分を作るときは、同じ回答者のAとBを対応づけます。ここでは、1つの項目だけを取り出し、回答者を行、刺激AとBを列にした横長表を作っています。

BマイナスAという方向を決めておくと、差分の符号を解釈しやすくなります。仮説がAの方が高いという場合でも、差分方向はレポート内で統一してください。

Python：差分の記述統計

diff = wide["diff_B_minus_A"].dropna()
diff_summary = {
    "n": diff.count(),
    "mean_diff": diff.mean(),
    "sd_diff": diff.std(),
    "median_diff": diff.median(),
    "q1": diff.quantile(0.25),
    "q3": diff.quantile(0.75),
}

diff_summary

必ず見ること

  • 平均差は何点か
  • 差分の標準偏差は大きいか
  • 中央値と平均差は近いか
  • 差分が極端な人はいないか

刺激AとBの比較では、差分の記述統計が重要です。平均差だけでなく、差分のばらつき、中央値、四分位範囲を確認します。

同じ平均差でも、ほとんどの人が少しずつBを高く評価している場合と、一部の人だけが大きくBを高く評価している場合では、意味が変わります。

今回の基本検定：対応のあるt検定

今回の標準デザインでは、

  • 同じ回答者が刺激Aと刺激Bを評価する
  • 各項目で B − A の差分を作れる

したがって、まず考えるのは

対応のあるt検定

何を検定しているか

B − A の平均差が、0と見なせるかを調べる。

今回の標準デザインでは、同じ回答者がAとBを評価するため、対応のあるt検定が基本になります。これは、A群とB群を別々の人として比べる方法ではありません。

対応のあるt検定では、回答者ごとのBマイナスAの差分を作り、その平均差が0と見なせるかを調べます。

t検定だけで終わらない

p値だけでは、結果を十分に説明できない。

合わせて報告するもの

見るもの 意味
AとBの平均・標準偏差 各刺激の評定の概要
平均差 B − A 刺激差の向きと大きさ
95%信頼区間 平均差の不確実さ
効果量 差の大きさの目安
分布図 値の偏り・外れ値・個人差

「有意だった」だけでは、レポートとして弱い。

p値は、結果の一部にすぎません。レポートでは、AとBの平均、標準偏差、平均差、信頼区間、効果量、図を合わせて示します。

特に授業内の小規模データでは、p値だけで強い結論を出すのではなく、図と効果量を見て、どの程度の差がありそうかを丁寧に述べます。

効果量：差の大きさを読む

対応あり比較では、差分を使った効果量を考える。

授業内での簡易的な目安

$$ d_z = \frac{\text{差分の平均}}{\text{差分の標準偏差}} $$

読み方

  • p値：差が0と見なせるかの手がかり
  • 効果量：差がどのくらい大きいかの手がかり
  • 図：その差が全体的な傾向か、一部の人の影響かを見る手がかり

対応あり比較では、回答者ごとの差分を使って効果量を計算できます。ここでは、差分の平均を差分の標準偏差で割る簡易的な効果量を紹介します。

効果量は、p値とは違い、差の大きさを読むための指標です。ただし、効果量だけでも判断せず、図や研究質問と合わせて解釈します。

差分が大きく歪むとき

差分の分布が強く歪む、外れ値の影響が大きい、nが小さい場合

⇒ Wilcoxon符号付順位検定 を検討

注意

  • これは対応あり2条件の代替法
  • 平均差を直接検定する方法ではない
  • 結果は、中央値・四分位範囲・図と一緒に読む

避けたいこと

  • Shapiro–Wilkのp値だけで機械的に切り替える
  • 都合のよい結果になる検定を後から選ぶ

差分の分布が大きく歪む場合は、Wilcoxon符号付順位検定を検討します。ただし、これは平均差を直接検定する方法ではなく、順位に基づく方法です。

また、正規性検定のp値だけで機械的にt検定かWilcoxonかを決めるのは避けます。まず可視化し、外れ値や差分の形を見て、分析理由を説明できるようにします。

Python：対応のあるt検定

from scipy import stats
a = wide["A"].dropna()
b = wide["B"].dropna()

# 同じ回答者だけにそろえる
paired = wide[["A", "B"]].dropna()

result = stats.ttest_rel(paired["B"], paired["A"])

print(result)

解釈の方向

  • B − A が正：Bの方が高い
  • B − A が負：Aの方が高い
  • p値だけでなく、平均差と図を必ず見る

Pythonでは、scipyのttest_relを使うと対応のあるt検定を実行できます。ここでは、同じ回答者についてAとBが両方そろっている行だけを使います。

出力されたp値だけを見て結論を書くのではなく、平均差、効果量、分布図と合わせて解釈します。

Python：Wilcoxon符号付順位検定

from scipy import stats
paired = wide[["A", "B"]].dropna()

result = stats.wilcoxon(
    paired["B"],
    paired["A"]
)

print(result)

使う場面

  • 差分の分布が強く歪んでいる
  • 外れ値の影響が大きい
  • 平均差より、順位にもとづく比較として報告したい

Wilcoxon符号付順位検定は、対応あり2条件のノンパラメトリックな代替法です。授業内では、t検定とWilcoxonの両方を機械的に並べるのではなく、どちらを主として読むかを分析理由とともに示すことを重視します。

分布が強く歪んでいる場合や外れ値が目立つ場合には、Wilcoxonの結果を補助的に示すことがあります。

3項目の結果を1枚の表にまとめる

VAS項目 A平均 B平均 平均差 B−A 95% CI 効果量 p値 読み取り
Q1 好ましさ
Q2 落ち着き
Q3 活発さ

読み方

  • 主分析項目を最初に読む
  • 残り2項目で結果の意味を補う
  • 予想と違う項目を考察に回す

3項目の結果は、1枚の表にまとめると読みやすくなります。A平均、B平均、平均差、信頼区間、効果量、p値、簡単な読み取りを入れます。

ただし、3項目をすべて同じ強さの結論として扱うのではなく、第12回で決めた主分析項目を最初に読みます。残りの項目は、結果の解釈を補う材料として使います。

2刺激 × 3項目でよくある結果パターン

パターン 例 解釈の方向
1項目だけ差が大きい 活発さだけBが高い 刺激差は特定の感性に出た
3項目すべて同じ方向 Bが全項目で高い 全体的にBが強い印象を与えた
項目で方向が分かれる Bは活発だが、Aは落ち着く 刺激の特徴が分化している
平均差は小さいが分散が大きい 人によって好みが割れる 個人差・刺激解釈の違いに注目

2刺激3項目の結果には、いくつか典型的なパターンがあります。1項目だけ差が出る場合、刺激の違いが特定の感性にだけ効いている可能性があります。

項目によって方向が分かれる場合は、刺激AとBの特徴が分化していると考えられます。平均差は小さいがばらつきが大きい場合は、個人差や刺激の解釈の違いを考察します。

余力があれば：3項目の関係を見る

今回の主分析は、刺激A・Bの差である。
ただし補助的に、次を見ることはできる。

例

  • 好ましさが高い人は、落ち着きも高く評価したか
  • 活発さが高い刺激は、好ましさも高い傾向か
  • B−A の差分どうしに関係があるか

注意

  • nは約24名で大きくない
  • 相関は因果を示さない
  • 主分析ではなく、考察の補助として扱う

3つのVAS項目の関係を見ることもできます。ただし、今回の主分析はあくまで刺激Aと刺激Bの比較です。相関は補助分析として扱います。

たとえば、好ましさの差分と落ち着きの差分が同じ方向に動くかを見ると、刺激のどの特徴が好ましさに結びついていそうかを考える手がかりになります。ただし、相関から因果を断定しないようにします。

結果の書き方：最小テンプレート

対応のある比較の例

Q2「落ち着き」について、刺激Aの平均は 68.2、刺激Bの平均は 54.7 であった。回答者ごとの差分 B−A の平均は −13.5 点であり、刺激Aの方が落ち着きが高い傾向が見られた。対応のあるt検定の結果、差は … であった。
効果量は … であった。

必ず含めること

  • どの項目か
  • AとBの平均
  • 差分の向き
  • 図と合わせた解釈

結果を書くときは、まずどの項目について述べているのかを明確にします。そのうえで、AとBの平均、差分の向き、検定結果、効果量を示します。

例文中の数値はあくまで形式の例です。実際のレポートでは、自分のグループのデータに基づいて書きます。

避けたい書き方

弱い例

  • Bの方が良かった
  • 有意差があったのでBが優れている
  • p値が有意でなかったので差はない
  • Aは落ち着く刺激であると証明された

修正の方向

  • どのVAS項目かを書く
  • 平均差の向きと大きさを書く
  • 図で分布を確認する
  • 「この2刺激の比較では」と範囲を限定する
  • 有意でなくても、効果量と不確実さを見る

レポートでは、「良かった」「優れている」といった総合評価だけで終わらせないようにします。今回測っているのは、3つのVAS項目であり、それぞれの強さです。

また、今回の結果だけで刺激一般の性質を証明することはできません。この2刺激、この回答者、この条件ではどうだったか、という範囲を明確にして書きます。

考察で必ず触れること

授業内プロジェクトには限界がある。

例

  • 回答者は授業内の学生に限られる
  • 回答数は約20〜24名程度である
  • 刺激A・Bは、比較したい要因以外も違う可能性がある
  • 順序効果や疲労の影響が残る可能性がある
  • 属性を原則取得しないため、属性別解釈はできない
  • VASは主観評価であり、行動や生理指標ではない

限界を書くことは、失敗を書くことではない。
結論の範囲を正確に示すことである。

考察では、結果の意味だけでなく限界も書きます。これは失敗を認めるためではなく、結論の範囲を正確に示すためです。

特に今回の授業内データでは、回答者数が限られること、刺激の統制に限界があること、属性を原則取得しないことを押さえます。
